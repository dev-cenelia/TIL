TIL – Spread & Rest 정리
1. 스프레드(Spread)란?

배열 또는 객체를 펼치는 문법.
주로 다음과 같은 상황에서 사용된다.

배열이나 객체를 펼쳐서 함수 인자로 전달할 때

얕은 복사를 할 때 (겉 객체만 새로 생기고 내부 객체는 참조 공유)

배열/객체를 자연스럽게 합칠 때

예시
const arr = [1, 2, 3];
const copy = [...arr];
const merged = [...arr, 4, 5];
Math.max(...arr);

얕은 복사 관련해서 이해한 점

스프레드로 만든 객체는 "겉 객체"는 독립적이지만,
그 안에 들어 있는 객체나 배열(참조값)은 그대로 공유한다.

const original = { a: 1, b: { x: 10 } };
const copy = { ...original };

copy.b.x = 999;
console.log(original.b.x); // 999


a는 숫자(원시값)라 독립적으로 복사됨

b는 객체라 주소가 그대로 복사됨

그래서 스프레드는 1-depth만 새롭게 만들어지는 얕은 복사

2. 레스트(Rest)란?

남는 값들을 하나의 변수(배열 또는 객체)로 모아 담는 문법.
스프레드가 값을 펼치는 거라면, 레스트는 반대로 값을 모으는 동작을 한다.

사용되는 주요 상황:

함수에서 들어오는 인자의 개수를 제한하지 않고 받고 싶을 때

구조분해 할당에서 필요한 것만 뽑고 나머지를 정리해서 보관하고 싶을 때

리액트 컴포넌트에서 props 중 특정 값만 다루고 나머지를 그대로 넘길 때

배열 구조 분해
const arr = [1, 2, 3, 4, 5];
const [first, second, ...rest] = arr;

console.log(first, second); // 1, 2
console.log(rest);          // [3, 4, 5]

객체 구조 분해
const user = { name: "Tom", age: 20, city: "Seoul" };
const { name, ...info } = user;

console.log(name);  // Tom
console.log(info);  // { age: 20, city: "Seoul" }

실무에서 주로 쓰는 이유

오늘 질문하면서 이해한 내용 정리:

API 응답에서 필요한 값만 꺼내고
나머지는 하나의 변수(rest)에 정리해서 나중에 사용할 수도 있도록 보관하는 용도

props 처리할 때 children 등 필요한 값만 쓰고
나머지는 rest로 모아 그대로 전달

데이터 전처리 시 앞부분/뒷부분을 나누기 쉬움

3. 스프레드 vs 레스트 비교
문법	역할	방향
스프레드 (...)	펼치기, 복사, 합치기	밖으로 퍼짐
레스트 (...)	남은 값을 하나로 모으기	안으로 모임

같은 문법처럼 보이지만
스프레드는 값을 밖으로 펼치고,
레스트는 값을 하나의 변수 안에 모은다.
